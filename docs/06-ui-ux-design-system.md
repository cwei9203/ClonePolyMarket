## 1. 设计理念

**「冷静的金融终端 + 娱乐的博弈快感」。** 不是模仿 Polymarket 的品牌，而是建立自有识别度。核心张力：

- **可信的数据感**：概率、走势、盈亏——像专业交易终端一样清晰、密集、精确。
- **克制的愉悦**：下注成功、结算获胜等关键时刻用动效奖励，但日常界面安静不喧闹。
- **反 AI 套版**：拒绝千篇一律的「卡片网格 + 每段小标签 eyebrow + 渐变文字」。见 §8。

### 1.1 场景句（决定主题）

> 用户在通勤地铁上、深夜床头刷手机，昏暗环境下快速扫一眼概率、下个注、看排名。

→ 这句话导向**深色主题为默认**：暗光环境友好、数据高亮更聚焦、贴合「金融终端」气质。提供浅色主题作为可选。

## 2. 色彩系统（OKLCH）

### 2.1 色彩策略：Committed（承诺型）

一个饱和品牌色承载识别度，中性色构成大面积背景，语义色专用于涨跌/胜负。**不做**满屏渐变，**不做** cream/sand 暖白背景（AI 套版重灾区）。

### 2.2 Token 定义

> 采用三层 Token 架构：primitive（原始值）→ semantic（语义角色）→ component（组件专用）。以下为 semantic 层，深色主题为主值。

```css
:root[data-theme="dark"] {
  /* 背景：近黑带极轻品牌色偏移，非纯黑（更柔和、有质感） */
  --bg:            oklch(0.18 0.015 265);   /* 页面底 */
  --surface:       oklch(0.22 0.018 265);   /* 卡片/面板 */
  --surface-raised:oklch(0.26 0.02  265);   /* 悬浮/弹窗 */
  --border:        oklch(0.32 0.02  265);   /* 分隔线 */

  /* 文字：高对比，正文绝不用浅灰（AI slop 头号问题） */
  --ink:           oklch(0.97 0.005 265);   /* 主文字，对 --surface ≥ 12:1 */
  --ink-secondary: oklch(0.78 0.01  265);   /* 次要文字，对 --surface ≥ 4.5:1 */
  --ink-muted:     oklch(0.62 0.01  265);   /* 弱化文字，仅用于大字/非正文 */

  /* 品牌主色：电光青紫（区别于 Polymarket 蓝，建立自有识别） */
  --brand:         oklch(0.68 0.19 285);
  --brand-ink:     oklch(0.98 0.01 285);    /* 品牌色上的文字 */

  /* 语义：涨跌 / Yes-No / 胜负 */
  --yes:           oklch(0.72 0.17 155);    /* Yes / 上涨 / 获胜：绿 */
  --yes-soft:      oklch(0.72 0.17 155 / 0.14);
  --no:            oklch(0.66 0.20 25);     /* No / 下跌 / 失败：红 */
  --no-soft:       oklch(0.66 0.20 25 / 0.14);
  --warning:       oklch(0.80 0.15 85);     /* 赔率过期/警示：琥珀 */

  /* 状态 */
  --focus-ring:    oklch(0.68 0.19 285 / 0.6);
}
```

> **对比度铁律**：正文对背景 ≥ 4.5:1，大字 ≥ 3:1，占位符同正文标准。上述 `--ink`/`--ink-secondary` 已按此设定；`--ink-muted` 仅用于 ≥18px 或非关键信息，绝不用于正文。

### 2.3 涨跌/胜负色语义

| 语义 | Token | 用途 |
|---|---|---|
| 上涨 / Yes / 获胜 / 盈利 | `--yes`（绿） | 价格上行、Yes 按钮、正收益 |
| 下跌 / No / 失败 / 亏损 | `--no`（红） | 价格下行、No 按钮、负收益 |
| 中性 / 未定 | `--ink-secondary` | 持平、待结算 |
| 警示 | `--warning` | 赔率过期、市场即将关闭 |

> 色盲友好：胜负不仅靠颜色，同时用 ↑/↓ 图标、正负号、文字标签冗余表达。

## 3. 排版

**字体配对**（在对比轴上配对，不配两个相似无衬线体）：

| 角色 | 字体 | 用途 |
|---|---|---|
| **数据 / 数字** | `Geist Mono` / `JetBrains Mono`（等宽） | 价格、概率、盈亏、余额——等宽保证数字对齐、终端感 |
| **界面文字** | `Geist` / `Inter`（几何无衬线） | 标题、正文、按钮、导航 |

- 数字全部用**等宽字体 + 表格数字**（`font-variant-numeric: tabular-nums`），让排行榜、盈亏列纵向对齐。
- 正文行长 65–75ch。标题 `text-wrap: balance`，长描述 `text-wrap: pretty`。
- 显示标题 clamp 上限 ≤ 6rem；字距下限 ≥ -0.04em。

**字号阶梯**（rem）：`0.75 / 0.875 / 1 / 1.125 / 1.25 / 1.5 / 2 / 2.5 / 3`。

## 4. 间距与布局

- **间距阶梯**（px）：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`，节奏有变化，非均匀堆叠。
- **1D 用 Flex，2D 用 Grid**；响应式网格 `repeat(auto-fit, minmax(300px, 1fr))`。
- **卡片克制使用**：市场列表用卡片是合理的（每个市场是独立可点实体）；但**绝不嵌套卡片**，盈亏/持仓表格用行而非卡片。
- **z-index 语义阶梯**：`dropdown(10) → sticky(20) → modal-backdrop(30) → modal(40) → toast(50) → tooltip(60)`，禁用 9999 魔法数。

## 5. 组件清单

> 基础组件参考 shadcn/ui（Radix + Tailwind）风格：可访问、可组合。以下为本项目专用组件。

| 组件 | 说明 | 关键状态 |
|---|---|---|
| `MarketCard` | 市场列表卡片：配图、问题、Yes/No 当前概率条、成交量、截止时间 | hover、即将截止、已结算 |
| `ProbabilityBar` | Yes/No 双色概率条，宽度映射概率 | 动画过渡（价格更新时平滑补间） |
| `PriceChart` | 历史赔率走势图（折线），可切 `1h/1d/1w/1m/max` | loading、无数据、悬停 tooltip |
| `TradePanel` | 下注面板：选 Yes/No、输入积分、显示预计份额与潜在赔付 | 余额不足、赔率过期、提交中 |
| `PositionRow` | 持仓行：市场、方向、份额、成本、市值、盈亏 | 盈/亏配色、待结算、已结算 |
| `BalancePill` | 顶栏余额/净值胶囊，数字变动时滚动动画 | 上升绿闪、下降红闪 |
| `LeaderboardRow` | 排名、头像、用户名、净值、ROI；当前用户高亮 | 前三名奖章、自己所在行置顶提示 |
| `StaleOddsBadge` | 「赔率更新中」提示（`syncedAt` 超阈值时） | — |
| `SettleToast` | 结算结果通知（获胜/失败），关键愉悦时刻 | 获胜彩带、失败低调 |

## 6. 动效

**原则**：动效是构建的一部分，服务于反馈与愉悦，非装饰。

| 场景 | 动效 | 曲线/时长 |
|---|---|---|
| 概率条更新 | 宽度平滑补间 | ease-out-quart, 400ms |
| 余额/净值变动 | 数字滚动 + 颜色闪烁（涨绿跌红） | ease-out, 500ms |
| 下注提交成功 | 按钮成功态 + 轻微缩放脉冲 | ease-out, 200ms |
| **结算获胜** | 彩带/粒子 + `SettleToast` 上浮 | 关键愉悦时刻，可稍隆重 |
| 结算失败 | Toast 低调淡入，不惩罚性动效 | crossfade, 300ms |
| 列表进入 | 逐项 stagger 淡入上移（每项适配，非统一反射） | ease-out, 50ms 间隔 |
| 走势图绘制 | 路径描绘（draw-in） | ease-out-expo, 600ms |

- **无 bounce/elastic**，用指数缓出。
- **`prefers-reduced-motion: reduce`**：所有动效退化为即时或纯淡入，非可选项。
- 揭示动效必须增强**已可见**的默认态，不得用类触发的过渡门控内容可见性（否则无头渲染/后台标签页会导致空白）。

## 7. 响应式

| 断点 | 布局 |
|---|---|
| `< 640px`（手机，主要场景） | 单列；市场卡片全宽；下注面板为底部抽屉 `<dialog>`；底部 Tab 导航 |
| `640–1024px`（平板） | 双列市场网格；下注面板侧滑 |
| `> 1024px`（桌面） | 三列市场网格；市场详情左图右面板；排行榜宽表 |

- 移动优先设计；下注抽屉用原生 `<dialog>`/popover 逃离 `overflow` 裁剪。
- 测试标题在各断点不溢出（长问题文本 + 大字号是溢出高发区）。

## 8. 反 AI 套版红线（Absolute Bans）

以下一律禁止，命中则重构：

- ❌ **侧边色条**（card 左/右 >1px 彩色 border 当装饰）。用完整边框或背景微染代替。
- ❌ **渐变文字**（`background-clip:text` + gradient）。用单一实色 + 字重/字号表达强调。
- ❌ **默认玻璃拟态**（decorative blur/glass）。除非罕见且有目的。
- ❌ **每段小标签 eyebrow**（每个 section 上方小号大写宽字距 kicker）。换其他节奏。
- ❌ **默认编号标记**（01/02/03 当脚手架）。仅真正的有序流程才编号。
- ❌ **千篇一律等大卡片网格**（icon+标题+文字无限重复）。
- ❌ **文字溢出容器**。每个断点测标题。

### AI slop 自检

> 若有人一眼断定「这是 AI 做的」，即失败。二阶自检：若能从「预测市场 + 非 Polymarket 蓝」推断出美学家族，说明落入第二层陷阱——本项目选**电光青紫深色终端**正是为跳出「预测市场=蓝白」的第一反射。

## 9. 可访问性

- 颜色对比达 WCAG AA（§2.2 已保证）。
- 胜负/涨跌不仅靠色彩（图标 + 符号 + 文字冗余）。
- 所有交互元素键盘可达，`:focus-visible` 有清晰 `--focus-ring`。
- 下注等关键操作有确认与可访问的状态播报（`aria-live` 用于余额/结算变动）。
- 走势图提供数据表降级/`aria-label` 描述趋势。
