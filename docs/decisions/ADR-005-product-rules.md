# ADR-005: 娱乐盘产品规则（账号、积分、市场范围、结算、排行榜）

## Status
Accepted

## Date
2026-07-09

## Context
ClonePolyMarket 是一个**娱乐性质**的 Polymarket 模拟盘。上一轮确定了「镜像真实赔率、不影响 Polymarket」（[ADR-002](./ADR-002-mirror-pricing.md)）后，仍有一批产品规则待拍板（原 [08 §4](../08-roadmap-and-open-questions.md#4-已确定的产品决策) 的开放问题 Q1–Q8）。这批规则直接决定数据模型、结算逻辑与整体交互，需在开工前一次性确定。

## Decision

| # | 规则 | 决策 | 理由 |
|---|---|---|---|
| **Q7** | 账号身份 | **昵称 + 密码**，昵称唯一去重，无需邮箱。随便开号，免注册门槛。 | 娱乐项目降低门槛；无邮箱即无找回，符合「用完可另开新号」的定位。 |
| **Q4/Q5** | 积分发放 | 注册发 **10,000**，**无任何补给/签到/破产重置**；用完即止。**无手续费**。 | 只发不补让排行榜公平可比（人人同一起跑线），且杜绝刷分；无费最简单最「爽」。 |
| **Q6** | 排行榜 | **仅按积分净值**单一口径（`balance + Σ未结算持仓市值`）排序。 | 积分只发不补，净值本身即公平基准，无需 ROI 等多口径。 |
| **Q1** | 市场范围 | **仅镜像二元（Yes/No）市场**，同步时过滤 `outcomes.length != 2`。多结果作 Post-MVP。 | 二元占 Polymarket 绝大多数；数据模型预留扩展，先跑通最划算。 |
| **Q8** | 市场展示 | **全量镜像**活跃二元市场，按**热度（volume）**排序。 | 无需人工运营；热度排序避免冷门市场淹没首屏。 |
| **Q3** | 特殊结算 | 50-50 / 作废市场按 **VOID 全额退回买入成本**处理。 | 无明确赢家时退钱比「按 0.5 比例赔付」更友好；此类市场极少，特殊分支成本可接受。详见 [05 §5.3](../05-trading-and-settlement.md#53-void-特殊结算全额退回)。 |
| **Q2** | 刷新方式 | **前端轮询**（15–30s），不引入 WebSocket/SSE。 | 数据本身有 30s 同步周期，实时推送收益低、复杂度高。 |

## 扩展预留（不牺牲当下简单性）
虽然 MVP 仅二元，数据模型按可扩展方式设计，**Post-MVP 放开过滤即可支持多结果，无需改表结构**：

- `position.outcomeIndex` / `trade.outcomeIndex` 用 `Int`（非布尔 `0|1`）。
- `market.outcomes` 存**变长数组**（MVP 恒为 2 项）。
- 结算逻辑按「获胜结果索引」而非硬编码 Yes/No。
- API 契约 `OutcomeIndex = number`（MVP 仅取 `0|1`）。

见 [02 §3.3](../02-data-model.md#33-positions用户持仓)、[03 §5.2](../03-polymarket-integration.md#52-仅镜像二元市场)。

## Alternatives Considered

### 积分可补给（签到 / 破产重置）
- Pros：提升留存，玩家破产后仍能回来。
- Cons：破坏排行榜公平性（起跑线不一），需把口径改为「累计发放」防刷分，复杂度上升。
- Rejected：与「只发不补、用完可另开号」的娱乐定位冲突；用户明确要求不补给。

### 多口径排行榜（净值 + ROI + 已实现）
- Pros：不同维度对比更丰富。
- Cons：只发不补时净值已是公平基准，多口径增加认知与实现成本。
- Rejected：单一净值口径足够，更简单直观。

### 首版即支持多结果市场
- Pros：市场覆盖更全（大选、体育冠军等热门多选）。
- Cons：下注面板、结算、UI 复杂度显著上升。
- Rejected：二元覆盖绝大多数；已做扩展预留，Post-MVP 平滑放开。

### 特殊结算按比例赔付（忠实镜像 0.5/0.5）
- Pros：完全忠实于 Polymarket。
- Cons：多数用户并非在 0.5 价位买入，比例赔付让其平白亏损，娱乐体验差。
- Rejected：VOID 全额退回对玩家更友好，且此类市场极少。

## Consequences
- `users` 无 `email` 字段；`username` 唯一约束即去重（见 [02 §3.1](../02-data-model.md#31-users)）。
- 无补给意味着**唯一积分发行来源是注册 +10,000**，简化守恒不变量 INV-1（见 [02 §4](../02-data-model.md#inv-1-积分守恒)）。
- 同步层新增二元过滤（见 [03 §5.2](../03-polymarket-integration.md#52-仅镜像二元市场)）。
- `markets` 新增 `voided` 字段承载 VOID 结算状态（见 [02 §3.2](../02-data-model.md#32-marketspolymarket-镜像)）。
- 排行榜与 API 去除 ROI/多口径，仅净值排序（见 [04 §6](../04-api-design.md#6-排行榜端点)）。
